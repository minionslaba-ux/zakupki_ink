# INK Flow — планировщик производства и закупок (Shtark INK)

Статическое веб-приложение (один `index.html` + данные). Бэкенд не нужен —
всё считается в браузере. Остатки склада, заявки и цены сохраняются в браузере
(localStorage) между сессиями. Для общей работы команды (1–3 человека) есть
синхронизация через облако — вкладка **«Облако»** (Firebase Firestore).

## Состав

```
index.html           — приложение (открывается прямо в браузере)
support.js            — рантайм
nev_data.json         — каталог сырья, полуфабрикатов, продукции и рецептур
catalog.json          — каталог сырья по категориям (закупки)
letterhead.png        — фирменный бланк (для печати/PDF/Word)
letterhead-data.js    — фирменный бланк, встроенный в base64 (офлайн-печать)
loader-logo-white.png — логотип для экрана загрузки
.nojekyll             — отключает обработку Jekyll на GitHub Pages
```

Для работы на GitHub Pages загрузите **все** файлы из этой папки в корень репозитория.

## Деплой на GitHub Pages

1. Создайте репозиторий и загрузите в него **содержимое этой папки** (файлы
   должны лежать в корне репозитория, либо в папке `/docs`).
2. В репозитории откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите источник **Deploy from a branch**,
   ветку `main` и папку `/ (root)` (или `/docs`, если положили файлы туда).
4. Сохраните. Через 1–2 минуты сайт будет доступен по адресу
   `https://<ваш-логин>.github.io/<репозиторий>/`.

### Через командную строку

```bash
git init
git add .
git commit -m "NEV calculator web"
git branch -M main
git remote add origin https://github.com/<логин>/<репозиторий>.git
git push -u origin main
```

Затем включите Pages в Settings → Pages (шаги 2–4 выше).

## Общая база для команды (облако)

По умолчанию данные у каждого пользователя свои (в его браузере). Чтобы
склад, заявки клиентов, закупки и цены были общими:

1. Создайте бесплатный проект на <https://console.firebase.google.com> →
   **Firestore Database → Create database** (режим production).
2. **Project settings** → ваше веб-приложение → скопируйте объект
   `firebaseConfig`.
3. В приложении откройте вкладку **«Облако»**, впишите своё имя, вставьте
   `firebaseConfig`, нажмите **«Подключить»**.
4. В Firestore → **Rules** разрешите доступ к коллекции `inkflow`. Для общего
   доступа по ссылке (без логинов): `allow read, write: if true;` — давайте
   ссылку только 1–3 доверенным сотрудникам.

Редактирует одновременно **один** человек (кнопка «Взять для редактирования»),
остальные видят изменения в реальном времени в режиме просмотра. Все правки
пишутся в **журнал изменений** с автором и датой. Блокировка снимается кнопкой
«Освободить» или автоматически через 2 минуты бездействия.

## Локальный запуск

Из-за `fetch()` данных просто открыть `index.html` с диска (`file://`) нельзя —
нужен локальный веб-сервер:

```bash
python -m http.server 8000
# затем откройте http://localhost:8000
```
