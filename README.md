# Shtark Flow — пакет для GitHub Pages

Самодостаточная сборка портала **Shtark Flow** (мультикомпанийный финансовый аналитик). Сборка: **2026-07-01** (метка видна в шапке портала).
Включены последние изменения:
- колонка **«% ввод»** во всех калькуляторах — десятичный ввод с запятой (15,64), диапазон 0,01–100,00 %;
- поле **«Тех. потери при производстве, %»** — расчёт себестоимости с учётом техотхода;
- **курс EUR/₽ берётся по ЦБ на сегодня** (поле нередактируемо), «Предполагаемый курс» остаётся ручным для сценарного расчёта.

## ⚠ ВАЖНО: доступ по секретному ключу
Портал защищён шлюзом. **Без ключа открывается фейковая страница «404 Not Found»** — это НЕ ошибка деплоя, а защита.
Открывать строго с ключом в адресе:

```
https://<логин>.github.io/<репозиторий>/?k=shtark-erp-2026
```

После первого входа ключ запоминается на сессию браузера. Если видите «404» — значит зашли без `?k=...`.
Ключ задаётся в исходнике `Калькулятор NEV.dc.html` (переменная `SECRET_KEY`) — при необходимости поменяйте и пересоберите.

## Что внутри
- `index.html` — весь портал в одном файле (HTML+CSS+JS инлайн, работает офлайн).
- `.gitignore` — защита ключей и боевых данных от попадания в репозиторий.
- `.nojekyll` — отключает обработку Jekyll на GitHub Pages (отдаёт файлы как есть).

## Деплой на GitHub Pages
1. Создайте репозиторий и положите туда содержимое этой папки (`index.html`, `.gitignore`, `.nojekyll`).
2. Settings → Pages → Source: `Deploy from a branch` → ветка `main`, папка `/ (root)`.
3. Через 1–2 минуты портал доступен по адресу `https://<логин>.github.io/<репозиторий>/?k=shtark-erp-2026`.
4. Если после обновления видите старую версию — жёсткий перезагруз: **Ctrl+Shift+R** (Cmd+Shift+R на Mac). Проверьте метку «сборка 2026-07-01» в шапке.

## Почему система не сломается
- **Данные отделены от кода.** Архив файлов, логи безопасности, задачи, маппинг, KPI, ключи AI хранятся в `localStorage` / `IndexedDB` / облаке (Firebase/Supabase) — НЕ в файлах репозитория. `git push` нового `index.html` их не затирает.
- **Ключи не в репозитории.** Серверные ключи (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `OCR_API_KEY`, service-account Firebase) — только в Environment Variables серверless-прокси (Cloudflare/Vercel/Netlify) или в защищённой админ-панели. Firebase web-config (клиентский apiKey) — не секрет, доступ ограничивают Security Rules.
- **`.gitignore`** не даёт случайно закоммитить `.env` и `serviceAccount*.json`.

## Обновление версии
Не редактируйте `index.html` вручную — это скомпилированный файл. Меняйте исходник `Калькулятор NEV.dc.html`, пересоберите автономную версию и замените `index.html`.

Полное руководство по деплою и архитектуре — в `DEPLOY.md` корня проекта.
