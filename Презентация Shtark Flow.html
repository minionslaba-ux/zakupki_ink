Что это

Финансовый «второй пилот» для завода

Shtark Flow собирает данные из 1С:КА, банка и почты, считает реальную себестоимость и маржу в двух валютах, находит аномалии и сам ставит задачи ответственным. Система работает на четырёх системных столпах.

ИИ-Коннект · почтовый робот

Документы приходят сами — по e-mail

Два входящих ящика → два контура учёта. Робот читает вложения, распознаёт их по алгоритму «парсить до победы» (до 5 итераций) и сам заносит данные в нужную компанию.
📨 Ящик №1 → Компания №1
Партионный учёт (Landed Cost). Списание по фактическим партиям закупки сырья с их уникальной полной себестоимостью.
📨 Ящик №2 → Компания №2
FIFO по выпускам. Списание по хронологии приходов 1С (First In, First Out), когда партионный учёт не ведётся.

Ядро управленческой отчётности

Себестоимость · гибридный учёт

📦

Партионный учёт

Дерево затрат из 1С: продукт → партия производства → аналитика затрат → сырьё + транспортные и доп. расходы.

⏱️

FIFO по хронологии

Где партий нет — списание First In, First Out по датам приходов; средневзвешенная СС 1 кг на дату выпуска.

🎯

Один источник истины

Маржа, рентабельность, ценообразование и план-факт берут СС только отсюда — за совпадающий период.

🗂️

Архив по периодам

Загрузки хранятся структурно (год → месяц); повторная загрузка месяца обновляет данные, а не плодит дубли.

Мульти-ИИ оркестратор

Сложную аномалию система эскалирует во внешнюю нейросеть — OpenAI GPT, Anthropic Claude, DeepSeek или OCR-сервис — с риск-скорингом и точками роста.

🎯

Эскалация по сложности

Внешний ИИ зовётся только на многофакторные, нелинейные, кросс-модульные аномалии — не на каждую мелочь.

🔐

Ключи не в коде

API-ключи — в Environment Variables серверless-прокси или GitHub Secrets; в браузер и репозиторий не попадают.

💾

Данные переживают деплой

Боевая база и настройки админа — в облаке (Firebase). Обновление кода на GitHub Pages их не затрагивает.

Разделение кода и данных · подробности в DEPLOY.md (архитектура, Env Vars, GitHub Secrets)
